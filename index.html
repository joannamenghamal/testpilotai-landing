Technical FAQ

Where does TestPilotAI run?

Checkout simulations run against your live production storefront. The system validates real-world behavior after theme, app, or payment changes ‚Äî not staging-only assumptions.

Does it execute real transactions?

No. The engine validates deterministic checkout signals without completing live payments.

How is this different from Cypress or Playwright tests?

Traditional test suites typically run in staging. TestPilotAI continuously validates production checkout flows and detects regressions that only surface in live storefront conditions.

What signals are evaluated?

The engine combines deterministic signals (URL state, cart integrity, variant confirmation) with weighted structural scoring across critical checkout elements.

What happens when a regression is detected?

Each run generates structured output including status (pass, degraded, fail), confidence score, root cause classification, and screenshot + HTML artifacts for debugging.

Is CI/CD integration supported?

Early access focuses on continuous production validation. API-triggered runs and CI integrations are planned based on team feedback.
